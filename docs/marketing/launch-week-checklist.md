# Nexphoria Launch Week Checklist

A tactical, day-by-day execution plan for the first 7 days of Nexphoria's official public launch. Assumes the site is built, products are live, and initial marketing assets are ready.

---

## Pre-Launch Day (Day 0) — Final Go/No-Go Checks

### Technical Verification
- [ ] Run full site audit: all product pages load, no 404s, no broken images
- [ ] Verify checkout flow end-to-end: add to cart → payment → order confirmation email fires
- [ ] Test mobile checkout on iOS Safari and Android Chrome
- [ ] Confirm all COA links return valid documents (per product and batch)
- [ ] Verify SSL certificate is valid (padlock shows in browser)
- [ ] Test site speed: Lighthouse performance score ≥ 80 on mobile
- [ ] Confirm Cloudflare Pages deployment is live and DNS propagated
- [ ] Verify sitemap.xml is accessible at nexphoria.com/sitemap.xml
- [ ] Submit sitemap to Google Search Console
- [ ] Verify Google Analytics (or Plausible) tracking is firing on all pages
- [ ] Test all contact/support forms
- [ ] Verify order confirmation emails render correctly (plain text + HTML)
- [ ] Check all product images are high-res, consistent format

### Inventory and Fulfillment
- [ ] Confirm opening inventory counts are accurate in the system
- [ ] Verify fulfillment partner (or in-house) is ready for Day 1 orders
- [ ] Cold-chain packaging materials on hand: insulated mailers, cold packs
- [ ] Confirm labeling: all vials have proper "Research Use Only" labels, batch numbers, expiration dates
- [ ] COA documentation system: every product batch has a retrievable COA ready for customer-facing access

### Marketing Assets
- [ ] Launch announcement email drafted and tested in ESP (subject line, preview text, mobile render)
- [ ] Hero social post designed (Instagram, X/Twitter, Reddit)
- [ ] Google Ads campaigns set to launch — budgets confirmed, final review done
- [ ] Meta Ads (if running) — creatives approved, campaign live-check done
- [ ] Launch blog post published and indexed
- [ ] Press release distributed to peptide research community outlets
- [ ] Influencer/researcher outreach: confirm any planned launch-day posts

## Emergency Protocols

### Site Goes Down
1. Check Cloudflare Pages status dashboard
2. Check GitHub repo for failed deployment
3. Roll back to last stable deployment via Cloudflare Pages dashboard
4. Post maintenance notice on social media

### Payment Processing Failure
1. Check Stripe dashboard for error details
2. Verify webhook endpoints are receiving events
3. Manually process affected orders if needed
4. Communicate with affected customers within 1 hour

### Inventory Stockout
1. Remove product from storefront (set to out of stock, do not delete)
2. Add "back in stock" email capture to product page
3. Notify customers who had the product in their cart
4. Expedite reorder with supplier
5. Post estimated restock date publicly

### Quality Issue / Customer COA Concern
1. Pull the specific batch COA immediately
2. Verify against the external lab report on file
3. If documentation matches: provide full documentation to customer
4. If any discrepancy exists: halt sales of that batch, initiate vendor investigation
5. Issue replacement or refund to affected customer; do not wait for investigation completion
